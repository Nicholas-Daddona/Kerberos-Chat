Kerberos-Chat

In our Kerberos-Chat implementation, the first thing you must do is enter your username and password

During the pause, the frontend will receive a salt from the server based upon the username, then generate a hashed password based upon that


Next the frontend will take that generated data to form the first authentitcator and send it to the server

If the server approves, the user will be sent to the messaging page and the server will send back a TGT and the session key
